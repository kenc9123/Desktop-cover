- 无边框深色桌面层窗口。

## 方向更新

用户认可第一版原型的动态效果，但明确当前方向不应以背景线场或应用快捷方式为核心。

新的目标：

- 中央应有一个圆形或类圆形设备核心。
- 圆盘需要随着鼠标移动转向不同方向。
- 圆盘上可以显示 CPU、GPU、内存占用、时间等系统信息。
- 应用快捷方式体系暂时后置。后期这个系统级桌面可能会做多个不同版本，用不同快捷方式应对不同需求。

实现决策：

- 将视觉重心改为中央系统核心。
- 暂时弱化 Dock 和快捷方式。
- 先用原生方式获取 CPU 和内存信息。
- GPU 先保留可见模块位和数据接口，等后续加入稳定的 GPU 读取方案。

## 第三轮方向更新

用户明确指出第二轮圆盘仍然像一层平面 HUD 皮，不像真正有体积的 3D 设备核心。

新的方向：

- 中央核心需要真实深度和透视，而不是 2D 圆环。
- 整体使用黑白工业 UI 语言。
- 更接近终末地式机能科幻界面气质，但不直接复用专有图标或资产。
- 快捷方式继续后置。
- 优先强化层叠机械核心、明确轮廓、黑白材质和绑定在核心上的数据读数。

- 根据鼠标位置旋转整个 3D 结构。
- 将强调色改为黑、白、灰。

## 第四轮方向更新

用户明确指出，时间、CPU、GPU、内存等信息不能作为独立 2D 浮层悬在圆盘上，而应该贴在 3D 圆盘上，随着圆盘一起产生姿态变化。

新的目标：

- 时间、CPU、GPU、内存、音频模块都需要跟随圆盘的 3D 姿态一起运动。
- 增加系统音频监听。
- 播放音乐时，尽量显示当前歌曲信息和专辑封面。
- 专辑封面模块不能是普通图片框，需要有更强的动画表现，最好能被音频电平驱动。

实现计划：

- 在同一个 `Viewport3D` 中使用 `Viewport2DVisual3D` 作为贴附在 3D 空间中的信息面板。
- 让这些信息面板和机械核心共享同一组 3D transform。
- 增加原生 Core Audio 电平监听。
- 增加媒体会话监听，在 Windows 暴露信息时读取标题、歌手和专辑封面。
- 让专辑模块根据音频电平做缩放、旋转和边框闪动。
