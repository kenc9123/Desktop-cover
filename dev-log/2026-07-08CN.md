# 2026-07-08 开发日记

## 目标

回应全屏反馈：中间机械圆盘在全屏状态下显得偏小，圆盘表面的信息显示也偏小、阅读不够舒服。

## 范围

- 所有改动继续限制在 `F:\Workspace\Personal\TopoShell` 内。
- 保留现有 WPF 3D 圆盘结构。
- 不重做 UI 概念，只优化全屏尺寸和盘面信息可读性。

## 改动

- 将原本固定 `660x620` 的核心舞台包进带上限的 `Viewbox`，让中间圆盘可以随全屏可用空间等比放大，同时在小窗口下仍能安全缩小。
- 稍微收窄 3D 相机视野并拉近相机，让实体圆盘在视口里占比更大。
- 放大贴在圆盘上的媒体、遥测和时间贴片：
  - 增大贴片的 3D 网格尺寸
  - 增大贴片内部视觉尺寸
  - 增大时钟、指标、媒体标题和标签字体
  - 增大遥测条和专辑封面仓
- 略微提高贴片背景不透明度和边框对比度，增强运动盘面上的可读性。
- 为生成的盘面文字启用 Display 文本排版和 ClearType 渲染。

## 底部 Dock 调整

用户提供全屏截图后指出，放大后的圆盘压到了下方区域，并希望下方 Dock 栏可以自定义放常用软件。

改动：

- 收窄核心舞台最大尺寸，并让它在中间区域向上对齐，给底部 Dock 留出稳定空间。
- 增加底部行高度，将原本的占位 footer 替换为真正的 Dock 条。
- 新增 `dock-shortcuts.json`，作为可编辑的软件快捷方式配置，并复制到应用输出目录。
- Dock 读取配置时做容错：JSON 不存在或损坏时使用内置默认快捷方式。
- Dock 按钮会尽量提取可执行文件图标；提取不到时显示文字首字母，并通过系统 Shell 启动目标。
- 保留右侧较小的 `CTRL+SPACE` 命令入口。

验证：

- `dotnet build src/TopoShell.App/TopoShell.App.csproj`
  - 结果：成功，0 个警告，0 个错误。
