# TopoShell 产品愿景

## 一句话定位

TopoShell 是一个 Windows 11 桌面交互层：它不替换系统内核，不重写 Explorer，而是在桌面上提供一套工业机能风的 Dock、启动器、状态层、视差背景和系统操作面板。

## 核心气质

关键词：工业机能风、简约不失复杂、低噪音、可互动、系统原生感。

它不应该像：

- 壁纸软件。
- 桌面挂件合集。
- 炫技型赛博霓虹皮肤。
- 网页 landing page。
- 传统开始菜单换皮。

它应该像：

- 一套贴在 Windows 合成器上的轻量设备操作台。
- 一个安静但有深度的桌面 HUD。
- 一个为工作、启动、切换、观察系统状态服务的界面层。
- 一个有光学层级、参数细节和微弱运动反馈的系统 UI。

## 设计原则

### 1. 简约不等于空

界面第一眼要干净，但近看要有结构：

- 细线网格。
- 分区编号。
- 状态刻度。
- 小型参数标签。
- 低透明度面板。
- 轻微发光边缘。
- 统一的间距和轴线。

复杂度应来自信息结构，而不是装饰堆叠。

### 2. 工业机能优先

所有视觉元素必须能解释为功能：

- 网格用于定位和视差。
- 线条用于分隔模块。
- 发光用于状态强调。
- 角标用于模块编号。
- 动效用于反馈系统状态。
- 视差用于表达层级，不用于眩晕式炫技。

### 3. 系统原生感

TopoShell 要像 Windows 11 的延伸，而不是遮在桌面上的网页：

- 尊重圆角、阴影、透明、Mica/Acrylic 的系统材质。
- 动效节奏接近系统级 UI，避免网页弹跳感。
- 默认不抢焦点、不挡任务、不干扰窗口操作。
- 所有系统修改都可退出、可恢复、可关闭。

### 4. 交互层级清晰

桌面层建议分为：

- Scene Layer：低亮度 3D/2.5D 网格、地形、视差背景。
- System Layer：Dock、状态栏、时间、电量、网络、音量。
- Command Layer：启动器、搜索、快捷命令、窗口切换。
- Detail Layer：设置、主题、插件、系统参数。

平时只显示 Scene + System，用户主动操作时才展开 Command 或 Detail。

### 5. 不替代 Windows，只增强 Windows

第一阶段不做真正的 Shell Replacement：

- 不替换 `explorer.exe`。
- 不改注册表 Shell。
- 不 hook 系统任务栏。
- 不安装驱动。
- 不改系统文件。

先做安全、可退出、可卸载、可恢复的桌面增强器。

## 目标用户体验

开机后用户看到的是一个干净的 Windows 桌面：

- 原任务栏可以自动隐藏或弱化。
- 底部或侧边有一个克制的 Dock。
- 屏幕边缘有低亮度状态信息。
- 鼠标移动时背景网格、面板和 Dock 有不同幅度的视差。
- 按快捷键呼出启动器，像命令面板一样启动应用、切换窗口或执行系统动作。
- 音频播放时，界面只发生微弱响应，不喧宾夺主。

## 视觉禁区

- 不做大面积紫蓝渐变。
- 不做满屏粒子和无意义光污染。
- 不做厚重卡片堆叠。
- 不做拟物按钮。
- 不做卡通化图标系统。
- 不让文字和控件像网页后台管理系统。

## 推荐默认视觉

- 背景：近黑色、低蓝绿色偏色。
- 主线条：低亮度青色。
- 强调色：青色、冷白、少量琥珀或红色警示。
- 字体：系统字体优先，数字和状态标签可使用等宽字体。
- 面板：8px 以内圆角，低透明度，细边框。
- 动效：150-300ms 以内，位移小，缓动稳。
