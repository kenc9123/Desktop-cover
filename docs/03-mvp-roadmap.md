# MVP 路线图

## 阶段 0：视觉原型

目标：验证工业机能风桌面层是否成立。

范围：

- 新建 .NET 桌面应用。
- 创建无边框透明主窗口。
- 嵌入 WebView2。
- 加载 Three.js 低亮度网格场景。
- 鼠标移动驱动轻微视差。
- 底部显示一个静态 Dock 原型。
- 右上角显示时间和系统状态占位。

验收标准：

- 界面第一眼简约，不像网页。
- 画面有深度，但不影响桌面使用。
- 鼠标视差细腻，不晕。
- Dock 不遮挡主要工作区域。
- 退出后不留下系统状态变更。

## 阶段 1：可用 Dock

目标：让 TopoShell 成为可日常试用的启动入口。

范围：

- 读取配置中的应用快捷方式。
- 点击启动应用。
- Dock 悬停放大。
- 图标右键菜单。
- 配置文件热重载。
- 托盘菜单：显示/隐藏、退出、打开设置。

验收标准：

- 用户能用 Dock 启动常用应用。
- Dock 动效稳定，无明显卡顿。
- 配置改动后无需重启。
- 所有功能可退出。

## 阶段 2：Command Layer

目标：提供一个比开始菜单更快的系统入口。

范围：

- 全局快捷键唤出命令面板。
- 搜索应用。
- 搜索预设命令。
- 快速打开路径。
- 切换主题。
- 切换 Scene/Dock 显示状态。

验收标准：

- 快捷键响应稳定。
- 搜索结果 100ms 内出现。
- 命令面板像系统 UI，不像网页弹窗。
- 关闭后焦点回到原应用。

## 阶段 3：系统状态与窗口感知

目标：让桌面层知道当前系统状态。

范围：

- 枚举运行中窗口。
- Dock 显示运行状态。
- 点击激活窗口。
- 获取音量、电池、网络状态。
- 状态栏显示简洁参数。

验收标准：

- Dock 能区分已运行和未运行应用。
- 点击运行中应用能正确激活窗口。
- 状态栏信息准确。
- 多显示器下不会错位。

## 阶段 4：音频响应与地形系统

目标：把 sonic-topography 的“音域回响”感觉转化为桌面层的弱响应效果。

范围：

- WASAPI loopback 捕获系统音频。
- FFT 拆分 8 个频段。
- Scene Layer 根据频段调整地形、线条、粒子和发光。
- 提供音频响应强度滑块。
- 电池模式下自动降低响应频率。

验收标准：

- 播放音乐时场景有呼吸感。
- 不播放音乐时场景仍然安静。
- 响应不抢用户注意力。
- 关闭音频监听后不保留设备占用。

## 阶段 5：任务栏弱化

目标：在用户明确开启后，降低原生任务栏存在感。

范围：

- 引导用户启用 Windows 自带自动隐藏任务栏。
- 提供 TopoShell 内的任务栏状态提示。
- 研究安全的任务栏透明/弱化方案。
- 不做 Explorer hook。

验收标准：

- 默认不修改任务栏。
- 用户开启后能恢复。
- 失败时不影响 Explorer。

## 第一版建议交付物

第一版不追求系统完整替换，只交付：

- 一个可运行的 TopoShell 原型。
- 一个工业机能风低亮度 3D 视差桌面层。
- 一个静态或半动态 Dock。
- 一个快捷键唤出的命令面板。
- 一个托盘退出入口。
- 一套主题配置。

## 项目风险

- 透明窗口和 WebView2 的合成性能可能不稳定。
- 多显示器、DPI、缩放比例会带来窗口定位问题。
- 任务栏相关能力容易触碰 Explorer 边界。
- 音频捕获需要谨慎处理设备权限和资源释放。
- 如果视觉过强，会从桌面增强器退化成壁纸软件。

## 当前决策

先做 `TopoShell`，不 fork `sonic-topography`。

原因：

- 目标不同：TopoShell 是桌面交互层，sonic-topography 是音乐可视化应用。
- 技术边界不同：TopoShell 需要系统集成，sonic-topography 主要是 Electron 应用窗口。
- 视觉可以参考：工业机能风、3D 网格、音频地形、细线面板、低亮度高密度信息。
- 系统能力需要重建：Dock、Command、窗口感知、任务栏弱化、全局快捷键。
